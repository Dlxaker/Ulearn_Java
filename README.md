В данном проекте реализована обработка данных о спортивных объектах из формата CSV, их хранение в реляционной базе данных и последующая визуализация.
Используемый стек технологий:
Java 24.
Maven для управления библиотеками.
SQLite JDBC для работы с базой данных.
JFreeChart для построения гистограмм.
Последовательность работы над проектом:
Создание моделей данных.
Разработаны классы Region и SportFacility для структурированного хранения информации. Приложение разделено на пакеты по логике работы: парсинг, база данных и аналитический сервис.
Реализация парсера.
Для чтения файла применены регулярные выражения. Это позволило корректно обрабатывать строки, где внутри кавычек содержатся запятые (например, в адресах) или переносы строк. Стандартные методы разделения строки здесь не использовались из-за сложности структуры данных.
Проектирование базы данных.
В SQLite созданы две таблицы. Для соблюдения 3-й нормальной формы названия регионов вынесены в отдельный справочник regions. Основная таблица facilities ссылается на регионы через внешний ключ. Запись данных производится пакетами для повышения производительности.
Аналитическая обработка.
Реализованы SQL запросы для выполнения задач по варианту:
Расчет среднего количества объектов в регионах.
Поиск трех регионов с наибольшим количеством объектов.
Объединение данных по Москве и Московской области в одну категорию с помощью оператора CASE прямо внутри запроса.
Визуализация данных.
На основе агрегированных данных из БД построена гистограмма. Подписи по оси регионов развернуты вертикально для исключения наложения текста. Итоговый график сохранен в файл chart.png.
Результаты работы проекта:
Проверка целостности данных.

Считано из CSV: 2855.

Инструкция по запуску:
Поместить файл data.csv в корень проекта.
Выполнить сборку через Maven.
Запустить класс Main.
